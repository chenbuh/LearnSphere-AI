# LearnSphere AI 关键模块维护地图

本文档不是功能说明书，而是“维护入口地图”。

适用场景：

- 想改某个模块，但不知道前台、后台、后端应该分别改哪里。
- 想排障，但不知道哪个控制器、服务、配置文件先看。
- 想评估改动风险，快速识别会波及哪些页面和数据表。

## 一、AI 治理

### 维护目标

- 管理系统提示词、AI 调用统计、稳定性、回路监控、A/B 实验和 AI 反馈审核。

### 前后台入口

| 位置 | 文件 | 说明 |
| --- | --- | --- |
| 后台路由入口 | `admin-vue/src/router/index.js` | `/ai` 对应 AI 治理总入口，`/ai-feedback` 对应 AI 反馈审计 |
| 后台页面壳层 | `admin-vue/src/views/AIGovernance.vue` | 页签壳层 |
| 后台编排层 | `admin-vue/src/composables/useAIGovernanceShell.js` | 页签映射、健康态透传、组件装配 |
| 后台页签配置 | `admin-vue/src/utils/aiGovernanceConfig.js` | 页签元数据、默认状态 |
| 后台接口封装 | `admin-vue/src/api/admin.js` | `/admin/ai/*`、`/admin/ai-tutor/*`、`/admin/ai/feedback/*` |

### 后端入口

| 层级 | 文件 | 说明 |
| --- | --- | --- |
| 控制器 | `backend/src/main/java/com/learnsphere/controller/admin/AdminAIController.java` | Prompt、日志、统计、健康、实验、简报 |
| 控制器 | `backend/src/main/java/com/learnsphere/controller/admin/AdminAITutorController.java` | AI 助教后台治理 |
| 控制器 | `backend/src/main/java/com/learnsphere/controller/admin/AdminAIFeedbackController.java` | AI 反馈审核与分析 |
| 学习者 AI 入口 | `backend/src/main/java/com/learnsphere/controller/AIGenerationController.java` | 阅读/写作/口语/语法等 AI 生成 |
| 学习者 AI 入口 | `backend/src/main/java/com/learnsphere/controller/AITutorController.java` | AI 助教会话 |
| 服务 | `backend/src/main/java/com/learnsphere/service/impl/AIGenerationServiceImpl.java` | AI 生成主逻辑、成本估算、降级逻辑 |
| 服务 | `backend/src/main/java/com/learnsphere/service/impl/AITutorServiceImpl.java` | AI 助教上下文与调用 |
| 服务 | `backend/src/main/java/com/learnsphere/service/impl/SystemPromptServiceImpl.java` | Prompt 读写、历史版本、回滚 |
| 服务 | `backend/src/main/java/com/learnsphere/service/impl/AIGenerationLogServiceImpl.java` | AI 调用日志入库与统计 |

### 数据与配置

- 实体：`AIGenerationLog`、`SystemPrompt`、`SystemPromptHistory`、`AIExperiment`
- 配置：`application.yml`、`application-prod.yml`、`application-secret.properties`
- 关联文档：[AI 成本控制](../AI功能/AI成本控制.md)、[观测链路实施](../性能优化/观测链路实施.md)

### 常见改动与落点

| 需求 | 优先修改位置 |
| --- | --- |
| 新增一个治理页签 | `aiGovernanceConfig.js` + `useAIGovernanceShell.js` + 新页签组件 |
| 新增 Prompt 类型 | `AdminAIController` + `SystemPromptServiceImpl` + 后台 Prompt 页签 |
| 统计口径变更 | `AIGenerationLogServiceImpl` / `AdminAIController` / Grafana 模板 |
| 调整模型或成本参数 | `AIGenerationServiceImpl` + 后端配置文件 |

### 维护风险

- Prompt `promptKey` 一旦改名，调用链和历史数据可能同时受影响。
- 成本和 Token 统计改动后，要同步核对后台页面和 Grafana 查询口径。
- A/B 实验逻辑改动后，至少要复核实验状态切换和流量分配结果。

| 词汇记录 | `frontend-vue/src/stores/vocabulary.js` | 词汇练习结果写入 |
| 语法记录 | `frontend-vue/src/composables/useGrammarPractice.js` | 语法练习记录写入 |
| 阅读记录 | `frontend-vue/src/composables/useReadingPractice.js` | 阅读结果写入 |
| 口语记录 | `frontend-vue/src/composables/useSpeakingPractice.js` | 口语结果写入 |

### 后台与后端入口

| 层级 | 文件 | 说明 |

### 常见改动与落点

| 需求 | 优先修改位置 |
| --- | --- |
| 新练习类型需要写学习记录 | 对应前台页面 / composable + `learningApi.createRecord` + `LearningRecordServiceImpl` |
| 调整复习策略 | `LearningRecordServiceImpl` 中复习时间和掌握度逻辑 |
| 调整分析报告来源 | `LearningRecordServiceImpl` + 学习分析相关接口 |
| 后台记录筛选条件扩展 | `AdminLearningContentController` + `Records.vue` |

### 维护风险

- 仓库里同时有 `LearningRecordController` 和 `LearningHistoryController` 两套入口，改动前先分清当前页面走哪条链路。
- 改 `contentType` 枚举时，要同步前台页面、后台筛选和数据库历史数据口径。
- 复习逻辑变更会直接影响学习分析和推荐结果。

## 三、权限控制

### 维护目标

- 保证学习者端、管理员端、VIP / AI 配额权限边界一致。

### 前后台入口

| 位置 | 文件 | 说明 |
| --- | --- | --- |
| 学习者登录页 | `frontend-vue/src/views/LoginView.vue` | 普通用户登录入口 |
| 学习者状态 | `frontend-vue/src/stores/user.js` | token 存储、登录 / 登出 |
| 学习者请求拦截 | `frontend-vue/src/utils/request.js` | `learnsphere-token` -> `satoken` |
| 学习者路由守卫 | `frontend-vue/src/router/index.js` | `requiresAuth`、维护模式检查 |
| 管理员登录页 | `admin-vue/src/views/Login.vue` | 管理员登录入口 |
| 管理员请求拦截 | `admin-vue/src/utils/request.js` | `admin-token` -> `satoken` |
| 管理员路由守卫 | `admin-vue/src/router/index.js` | 后台页面鉴权 |

### 后端入口

| 层级 | 文件 | 说明 |
| --- | --- | --- |
| 登录控制器 | `backend/src/main/java/com/learnsphere/controller/AuthController.java` | 用户登录与会话 |
| 登录控制器 | `backend/src/main/java/com/learnsphere/controller/AdminAuthController.java` | 管理员登录与会话 |
| 全局拦截 | `backend/src/main/java/com/learnsphere/config/SaTokenConfig.java` | 登录校验与后台角色校验 |
| 权限实现 | `backend/src/main/java/com/learnsphere/config/StpInterfaceImpl.java` | 权限列表实现 |
| 注解 | `backend/src/main/java/com/learnsphere/common/annotation/RequireVip.java` | AI / VIP 权限注解 |
| 切面 | `backend/src/main/java/com/learnsphere/common/aspect/VipCheckAspect.java` | 配额检查与扣减 |
| 注解 | `backend/src/main/java/com/learnsphere/common/annotation/AdminOperation.java` | 管理员操作审计 |
| 切面 | `backend/src/main/java/com/learnsphere/common/aspect/AdminLogAspect.java` | 后台操作日志记录 |

### 常见改动与落点

| 需求 | 优先修改位置 |
| --- | --- |
| 新页面需要登录保护 | 前端路由 `meta.requiresAuth` + 后端接口校验 |
| 某接口仅管理员可用 | 控制器放到 `/api/admin/**` 或显式加 `@SaCheckRole("admin")` |
| 新 AI 功能要计入配额 | 对应 Service / Controller 增加 `@RequireVip`，并核对配额 key |
| 调整登录态失效处理 | 前端 `request.js` + 后端会话配置 |

### 维护风险

- 当前前后台本地存储 key 不同：学习者端用 `learnsphere-token`，后台用 `admin-token`，不要混写。
- `/api/admin/**` 的保护依赖 `SaTokenConfig`，新增后台接口时不要绕开这层约束。
- `RequireVip` 改动会影响 AI 调用、计费感知和用户体验，必须同步核对前端提示。

## 四、词库导入与内容维护

### 维护目标

- 维护词汇题库来源、批量导入、AI 补全和后台编辑流程。

### 前后台入口

| 位置 | 文件 | 说明 |
| --- | --- | --- |
| 后台词汇页 | `admin-vue/src/views/Vocabulary.vue` | 词汇管理总入口 |
| 后台接口 | `admin-vue/src/api/admin.js` | 批量导入、AI 补全、去重、清理占位符 |
| 前台数据警告 | `frontend-vue/src/data/vocabulary.js` | 已明确静态词库加载停用，依赖后端数据 |
| 前台练习逻辑 | `frontend-vue/src/composables/useVocabularyPractice.js` | 读取词汇数据并触发学习记录 |

### 后端入口

| 层级 | 文件 | 说明 |
| --- | --- | --- |
| 控制器 | `backend/src/main/java/com/learnsphere/controller/AdminController.java` | 后台词汇 CRUD、批量导入、AI 补全 |
| 控制器 | `backend/src/main/java/com/learnsphere/controller/VocabularyManagementController.java` | 基于受控来源的批量导入 |
| 控制器 | `backend/src/main/java/com/learnsphere/controller/VocabularyImportController.java` | 预置词库导入入口 |
| 注册表 | `backend/src/main/java/com/learnsphere/config/VocabularyImportSourceRegistry.java` | 允许导入的安全来源 |
| 工具 | `backend/src/main/java/com/learnsphere/utils/VocabularyImporter.java` | JS 词库文件解析与入库 |
| 服务 | `backend/src/main/java/com/learnsphere/service/impl/VocabularyServiceImpl.java` | 词汇业务逻辑 |

### 常见改动与落点

| 需求 | 优先修改位置 |
| --- | --- |
| 新增一个可导入词库文件 | `VocabularyImportSourceRegistry` + 导入控制器入口 + 文档说明 |
| 调整导入字段映射 | `VocabularyImporter` |
| 后台补充 AI 词汇详情 | `AdminController` + 前台后台调用入口 |
| 导入后前台仍查不到数据 | 先查数据库，再查前台是否仍走旧静态数据路径 |

### 维护风险

- 导入来源已收口到 `VocabularyImportSourceRegistry`，不要在控制器里散落新的硬编码文件路径。
- 词库导入会直接影响前台学习体验，导入后至少要验证后台列表、前台词汇页和学习记录写入三处。
- 如果继续保留历史静态词库文件，只能作为离线素材，不应再视为主数据源。

## 五、建议维护顺序

当你准备改动某个模块时，建议固定按下面顺序过一遍：

1. 先确认页面入口和 API 封装是否已经定位。
2. 再确认后端控制器和服务是否只有一条生效链路，避免改到旧实现。
3. 最后检查配置文件、数据库实体和监控口径是否需要同步更新。

这样可以明显降低“页面改了、接口没改”或者“接口改了、统计口径没跟上”的回归风险。
