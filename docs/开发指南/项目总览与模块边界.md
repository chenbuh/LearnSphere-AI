# LearnSphere AI 项目总览与模块边界

本文档用于回答三个问题：

1. 这个仓库由哪些系统组成。
2. 每个系统的代码入口、运行入口和配置入口在哪里。
3. 哪些内容由前端内置，哪些由后端提供，哪些可以通过管理后台维护。

如果是首次接手项目，建议先阅读本文，再按需跳转到部署、后端架构、AI 治理等细分文档。

## 一、仓库结构总览

仓库当前主要由 5 个可识别层组成：

- `frontend-vue`
  面向学习者的前台应用，负责词汇、语法、听力、阅读、写作、模拟考试、学习分析等学习场景。
- `admin-vue`
  面向运营和管理员的后台应用，负责用户、内容、日志、系统配置、AI 治理、敏感词与 Redis 管理。
- `backend`
  Spring Boot 后端，负责鉴权、业务 API、AI 调用、配置落库、审计日志、监控指标暴露。
- `docs`
  项目文档中心，覆盖部署运维、开发指南、后端架构、业务功能、AI 功能、性能优化等。
- 根目录脚本与配置
  包括 `package.json`、`package-lock.json`、`scripts/本地发版准备.ps1`、若干 SQL 脚本和部署辅助文件。

补充目录说明：

- `gateway`
  当前仓库中存在，但不是这一轮交付的主运行入口。后续若启用网关，需要补单独的运行说明和路由边界文档。
- `node_modules`
  本地依赖目录，不属于版本交付内容。

## 二、系统职责划分

### 1. 前台 `frontend-vue`

职责：

- 提供学习者端页面和交互体验。
- 承担页面状态管理、练习流程编排、前端埋点和 Web Vitals 采样。
- 调用后端 API 获取题目、词汇、学习记录、AI 反馈等数据。

典型页面入口：

- `src/views/VocabularyView.vue`
- `src/views/GrammarView.vue`
- `src/views/ListeningView.vue`
- `src/views/ReadingView.vue`
- `src/views/WritingView.vue`
- `src/views/MockExamView.vue`
- `src/views/LearningHubView.vue`
- `src/views/LearningAnalysisView.vue`

运行入口：

- 开发：`npm run dev`
- 构建：`npm run build`

配置入口：

- [frontend-vue/.env.example](E:/Project/LearnSphere%20%20AI/frontend-vue/.env.example)
- 当前关键变量：`VITE_API_URL`

### 2. 后台 `admin-vue`

职责：

- 提供运营、内容维护和系统治理能力。
- 管理词汇、语法、写作、口语、内容审核、通知、系统设置、操作日志。
- 承接 AI 治理、反馈审核、敏感词审计、Redis 管理、系统监控等运维向界面。

典型页面入口：

- `src/views/Dashboard.vue`
- `src/views/Users.vue`
- `src/views/Content.vue`
- `src/views/AIGovernance.vue`
- `src/views/AITutorManagement.vue`
- `src/views/SystemSettings.vue`
- `src/views/SystemMonitor.vue`

运行入口：

- 开发：`npm run dev`
- 构建：`npm run build`

配置入口：

- [admin-vue/.env.example](E:/Project/LearnSphere%20%20AI/admin-vue/.env.example)
- 当前关键变量：`VITE_API_URL`

### 3. 后端 `backend`

职责：

- 暴露前后台共用 API。
- 负责用户、学习记录、题目内容、AI 请求、系统配置、审计日志、配额统计等核心业务。
- 通过 `Actuator + Prometheus` 暴露系统与 AI 观测指标。

运行入口：

- 开发：`mvn spring-boot:run`
- 打包：`mvn clean package`

配置入口：

- [backend/src/main/resources/application.yml](E:/Project/LearnSphere%20%20AI/backend/src/main/resources/application.yml)
- [backend/src/main/resources/application-prod.yml](E:/Project/LearnSphere%20%20AI/backend/src/main/resources/application-prod.yml)
- [backend/src/main/resources/application-secret.properties.example](E:/Project/LearnSphere%20%20AI/backend/src/main/resources/application-secret.properties.example)

核心说明：

- `application.yml` 用于基础配置与默认行为。
- `application-prod.yml` 用于生产环境覆盖项。
- `application-secret.properties` 或环境变量用于敏感信息注入，避免源码明文。

### 4. 文档 `docs`

职责：

- 作为交接、开发、部署、排障和演进的统一知识库。

## 三、资源归属边界

这部分是接手时最容易混淆的地方。

### 1. 前端内置资源

通常包括：

- 页面结构、组件布局、交互状态。
- 一部分练习流程默认配置、静态选项、展示文案。
- 前端埋点、UI 动画和页面级体验逻辑。

判断原则：

- 如果内容只影响界面展示，且不需要运营后台动态修改，优先留在前端。
- 如果内容要跟随版本发布，且不是运营数据，也优先作为前端代码或配置管理。

### 2. 后端提供资源

通常包括：

- 用户信息、学习记录、练习结果、考试记录。
- AI 请求结果、配额信息、日志、审计数据。
- 内容列表、题目数据、可持久化配置、监控指标。

判断原则：

- 只要需要跨端共享、需要落库、需要审计、需要统计，就应该由后端提供。

### 3. 管理后台可维护资源

当前主要包括：

- 用户状态和通知。
- 内容管理中的词汇、语法、口语、写作等可运营内容。
- 系统设置、配额设置、敏感词词库、AI 治理配置。
- 操作日志、用户日志、反馈审核等后台治理能力。

判断原则：

- 如果内容需要运营侧频繁修改，且不应该每次改动都走代码发版，就优先放到后台维护。

## 四、配置边界

### 1. 根目录配置

- [package.json](E:/Project/LearnSphere%20%20AI/package.json)
  负责 workspace、统一构建和本地发版脚本入口。
- [package-lock.json](E:/Project/LearnSphere%20%20AI/package-lock.json)
  锁定前后台 npm 依赖版本。

### 2. 前端配置

- `frontend-vue/.env*`
- `admin-vue/.env*`

主要负责：

- API 基础地址。
- 与构建环境相关的前端变量。

### 3. 后端配置

- `backend/src/main/resources/application*.yml`
- `backend/src/main/resources/application-secret.properties`
- Linux 宝塔中的环境变量或外置配置文件

主要负责：

- 数据库、Redis、AI 服务、邮件、日志、Actuator 暴露等服务级配置。
