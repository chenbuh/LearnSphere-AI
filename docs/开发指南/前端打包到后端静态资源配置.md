# 前端打包到后端静态资源配置说明

## 配置概述

当前项目采用"前端构建产物直接输出到后端静态资源目录"的方案，无需手动复制文件。

## 目录结构

```
backend/src/main/resources/static/
├── dist/                          # 用户前端构建产物（由 frontend-vue 自动输出）
│   ├── index.html                 # 用户前端入口
│   ├── assets/                    # 带 hash 的静态资源
│   │   ├── js/                    # JavaScript 文件
│   │   ├── css/                   # CSS 文件
│   │   └── [ext]/                 # 其他资源（图片、字体等）
│   ├── sw.js                      # Service Worker
│   ├── registerSW.js              # Service Worker 注册脚本
│   └── workbox-*.js               # Workbox 运行时
└── admin/                         # 管理后台构建产物（需手动同步）
    ├── index.html
    └── assets/
```

## 1. Vite 构建配置

**文件：** `frontend-vue/vite.config.js`

### 关键配置

```javascript
// 前端构建产物输出到后端静态资源目录
const frontendStaticOutDir = fileURLToPath(
  new URL('../backend/src/main/resources/static/dist', import.meta.url)
)

export default defineConfig({
  build: {
    // 直接输出到后端静态目录
    outDir: frontendStaticOutDir,
    emptyOutDir: false,  // 不清空整个目录（避免删除 admin）

    rollupOptions: {
      output: {
        // 文件命名规则（带 hash）
        chunkFileNames: 'assets/js/[name]-[hash].js',
        entryFileNames: 'assets/js/[name]-[hash].js',
        assetFileNames: 'assets/[ext]/[name]-[hash].[ext]'
      }
    }
  },

  plugins: [
    // 构建前清理旧的 dist 目录
    {
      name: 'clean-backend-frontend-static',
      apply: 'build',
      buildStart() {
        cleanFrontendStaticOutput()
      }
    }
  ]
})
```

### 构建命令

```bash
# 在项目根目录执行
npm run build:frontend

# 或在 frontend-vue 目录执行
cd frontend-vue
npm run build
```

构建完成后，产物会自动输出到 `backend/src/main/resources/static/dist/`

## 2. Spring Boot 静态资源配置

**文件：** `backend/src/main/java/com/learnsphere/config/WebConfig.java`

### 资源映射规则

```java
@Override
public void addResourceHandlers(ResourceHandlerRegistry registry) {

    // 1. 管理后台（优先级最高）
    registry.addResourceHandler("/admin", "/admin/", "/admin/**")
            .addResourceLocations("classpath:/static/admin/")
            .setCacheControl(CacheControl.noStore())
            .resourceChain(true)
            .addResolver(new PathResourceResolver() {
                // SPA 路由支持
            });

    // 2. 带 hash 的静态资源（长期缓存）
    registry.addResourceHandler("/assets/**", "/dist/assets/**")
            .addResourceLocations("classpath:/static/dist/assets/")
            .setCacheControl(CacheControl.maxAge(7, TimeUnit.DAYS).immutable());

    // 3. Service Worker（不缓存）
    registry.addResourceHandler(
            "/sw.js", "/registerSW.js", "/workbox-*.js",
            "/dist/sw.js", "/dist/registerSW.js", "/dist/workbox-*.js")
            .addResourceLocations("classpath:/static/dist/")
            .setCacheControl(CacheControl.noStore());

    // 4. dist 目录直出
    registry.addResourceHandler("/dist/**")
            .addResourceLocations("classpath:/static/dist/")
            .setCacheControl(CacheControl.noStore());

    // 5. 用户前端 SPA 路由（优先级最低）
    registry.addResourceHandler("/**")
            .addResourceLocations("classpath:/static/dist/")
            .setCacheControl(CacheControl.noStore())
            .resourceChain(true)
            .addResolver(new PathResourceResolver() {
                // 所有未匹配的路由回退到 index.html
            });
}
```

### 视图控制器配置

```java
@Override
public void addViewControllers(ViewControllerRegistry registry) {
    // 根路径映射
    registry.addViewController("/").setViewName("forward:/dist/index.html");
    registry.addViewController("/dist").setViewName("forward:/dist/index.html");
    registry.addViewController("/dist/").setViewName("forward:/dist/index.html");

    // 管理后台路径映射
    registry.addViewController("/admin").setViewName("forward:/admin/index.html");
    registry.addViewController("/admin/").setViewName("forward:/admin/index.html");
}
```

```

http://localhost:8080/assets/js/index-[hash].js

# 管理后台资源

# Service Worker
http://localhost:8080/sw.js
http://localhost:8080/dist/sw.js  # 两种路径都支持
```

## 4. 缓存策略

| 资源类型 | 路径 | 缓存策略 | 说明 |
|---------|------|---------|------|
| 带 hash 的资源 | `/assets/**` | `max-age=604800, immutable` | 7天强缓存，内容不变 URL 不变 |
| index.html | `/`, `/dist/index.html` | `no-store` | 不缓存，每次都获取最新 |
| Service Worker | `/sw.js`, `/registerSW.js` | `no-store` | 不缓存，确保及时更新 |
| 管理后台 | `/admin/**` | `no-store` | 不缓存 |
| 用户上传文件 | `/uploads/**` | `no-store` | 不缓存 |

## 7. 常见问题

### 问题 1：前端构建后，后端访问 404

**原因：** 构建产物未正确输出到 `backend/src/main/resources/static/dist/`

**解决：**
```bash
# 检查输出目录
ls backend/src/main/resources/static/dist/

# 如果目录不存在或为空，重新构建
npm run build:frontend
```

### 问题 2：静态资源路径 404

**原因：** WebConfig 路径映射配置错误

**解决：**
- 检查 `WebConfig.java` 中的 `addResourceHandlers` 配置
- 确保 `/assets/**` 和 `/dist/assets/**` 都映射到 `classpath:/static/dist/assets/`

### 问题 3：SPA 路由刷新后 404

**原因：** 缺少 SPA 路由回退配置

**解决：**
- 确保 `/**` 路径的 `PathResourceResolver` 正确配置
- 所有未匹配的路由应回退到 `index.html`

### 问题 4：Service Worker 未更新

**原因：** Service Worker 被缓存

**解决：**
- 确保 `/sw.js` 设置了 `Cache-Control: no-store`
- 清除浏览器缓存或使用隐私模式测试

### 问题 5：管理后台和用户前端冲突

**原因：** 路径映射优先级错误

**解决：**
- 确保 `/admin/**` 在 `/**` 之前注册
- 在 `/**` 的 resolver 中排除 `admin/` 路径

## 8. 最佳实践

### 开发阶段

1. 使用 Vite 开发服务器（热更新）
   ```bash
   npm run dev:frontend  # 用户前端
   npm run dev:admin     # 管理后台
   ```

2. 后端使用 IDE 启动或 `mvn spring-boot:run`

3. 前端通过代理访问后端 API（已在 `vite.config.js` 配置）

### 构建阶段

1. 先构建前端，确保静态资源正确输出
2. 再构建后端，将静态资源打包进 JAR
3. 验证 JAR 包中包含完整的静态资源

### 部署阶段

1. 上传 JAR 文件到服务器
2. 启动应用
3. 验证静态资源访问正常
4. 检查浏览器控制台无 404 错误

## 10. 总结

当前配置的优势：
- ✅ 前端构建自动输出到后端，无需手动复制
- ✅ 支持 SPA 路由（Vue Router history 模式）
- ✅ 合理的缓存策略（带 hash 的资源长期缓存）
- ✅ 支持多个前端应用（用户前端 + 管理后台）
- ✅ 开发和生产环境配置分离

注意事项：
- ⚠️ 管理后台需要手动同步到 `backend/src/main/resources/static/admin/`
- ⚠️ 构建前端后必须重新打包后端 JAR
- ⚠️ Service Worker 和 index.html 不能缓存
- ⚠️ 路径映射的注册顺序很重要（越具体的越先注册）
