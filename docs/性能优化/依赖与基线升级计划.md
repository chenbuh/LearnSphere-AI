# 依赖与基线升级计划（P3-8）

更新时间：2026-02-26  
适用范围：`backend`、`frontend-vue`、`admin-vue`、本地语音 Python 服务、CI 基线

## 1. 背景

后端升级起点为 `Spring Boot 3.0.5`，当前已完成第二跳到 `3.3.0`，前端生态已升级到较新的 Vite/Vue 版本，CI 基线与运行时基线存在潜在漂移。  
本计划用于分阶段完成“依赖升级 + 基线统一 + 可回滚发布”，控制升级风险并降低后续维护成本。

## 3. 升级目标

- 后端迁移到受支持的 Spring Boot 3.x 基线（分阶段推进，避免一次性跨越过大）。
- 统一 Java/Node/Python 运行时与 CI 基线，确保“开发机可跑 ≈ CI 可跑”。
- 建立依赖升级的验证矩阵、发布闸门与回滚方案，避免升级后回归不可控。

## 4. 分阶段实施计划

### 阶段 0：准备与冻结（0.5 天）

- 输出依赖清单（后端 Maven + 双前端 npm + 语音 Python）。
- 冻结当前可发布快照（代码 tag + 制品备份）。
- 建立升级分支（示例：`chore/dependency-baseline-upgrade`）。

交付物：
- 基线清单文档（本文件）。
- 升级任务清单（Issue/看板项）。

### 阶段 1：低风险升级与基线统一（1~2 天）

- 后端：
  - 先完成同代小版本与外围依赖清理（重复/过旧/冲突项）。
  - 补充 Maven 依赖审计命令（`mvn versions:display-dependency-updates`、`mvn dependency:tree`）。
- 前端：
  - 统一 `frontend-vue` 与 `admin-vue` 的关键依赖大版本策略（Vue Router、Axios、构建链）。
  - 在 CI 引入统一 Node LTS 基线（已落地 Node 20 LTS，后续评估 22）。
- 语音服务：
  - 增加 `requirements-voice.txt`，使注释依赖与启动脚本一致。

发布策略：
- 仅灰度到测试环境，不直接进生产。

### 阶段 2：Spring Boot 主干升级（2~3 天）

建议路径：
1. `3.0.5 -> 3.2.x`（已完成，当前为 `3.2.12`）。
2. `3.2.x -> 3.3.x`（已完成，当前为 `3.3.0`）。

重点检查：
- `spring.cache`、Redis、Actuator、Security/Sa-Token、MyBatis-Plus 兼容性。
- Knife4j/OpenAPI 与 Jakarta 相关依赖是否需要同步调整。
- 自定义配置类（缓存、异步、CORS、鉴权）的行为是否变化。

### 阶段 3：回归验证与生产灰度（1~2 天）

- 全量自动化：
  - `backend`: `mvn -q test`
  - `frontend-vue`: `npm ci && npm run build && npm test`
  - `admin-vue`: `npm ci && npm run build`
- 核心链路人工冒烟：
  - 登录鉴权（含限流）
  - 导入与后台管理接口
  - AI 调用降级/重试
  - 路由守卫与系统配置刷新
  - 语音 TTS/STT 可用性
- 灰度策略：
  - 先小流量/单节点，观察 30~60 分钟再扩大。

## 5. 验收门槛（DoD）

- 升级分支 CI 全绿（后端测试 + 双前端构建 + 前端测试）。
- 生产关键链路零阻断问题（登录、学习、AI、管理后台）。
- 启动失败率、5xx、平均延迟无显著劣化（阈值建议 ±10% 内）。
- 高危依赖漏洞较升级前显著下降（至少完成一次依赖安全扫描留档）。

## 6. 回滚方案

- 代码回滚：保留升级前 tag（示例：`baseline-pre-dep-upgrade`），异常时直接回切。
- 制品回滚：保留最近两个稳定版本的后端包与前端静态制品。
- 配置回滚：升级相关配置项采用显式开关，避免一次性替换。
- 数据回滚：本计划不引入强制 DDL；若伴随数据库改动，需独立准备 SQL 回滚脚本并先演练。

## 7. 风险清单与缓解

| 风险 | 描述 | 缓解措施 |
| --- | --- | --- |
| 版本跨越过大 | 一次升级过多导致定位困难 | 严格分阶段，单阶段可独立回退 |
| CI 与本地不一致 | 本地可跑但 CI 失败 | 锁定 Node/JDK 版本并写入工作流 |
| 间接依赖冲突 | 升级后出现运行时 `NoSuchMethodError` | 升级前后都输出依赖树并比对 |
| 语音链路漂移 | Python 依赖未锁定导致启动失败 | 固化 requirements 并纳入冒烟 |
