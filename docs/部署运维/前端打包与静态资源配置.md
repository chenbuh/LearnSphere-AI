## 概述
LearnSphere AI 采用**前后端合包**模式部署：Vue 前端打包后的 `dist/` 文件放入 Spring Boot 后端的 `src/main/resources/static/` 目录，由 Spring Boot 内置 Tomcat 直接提供静态文件服务，无需独立 Nginx。

```
frontend-vue/dist/*  →  backend/src/main/resources/static/
admin-vue/dist/*     →  backend/src/main/resources/static/admin/
```

## 目录结构

```
backend/src/main/resources/static/
├── index.html          ← 前端入口（SPA，不缓存）
├── favicon.svg
├── manifest.webmanifest
├── registerSW.js       ← PWA Service Worker 注册（不缓存）
├── sw.js               ← Service Worker（不缓存）
├── workbox-*.js        ← Workbox 运行时（不缓存）
├── robots.txt
├── pwa-192x192.png
├── pwa-512x512.png
├── assets/             ← 带 contenthash 的 JS/CSS/图片（7天缓存）
│   ├── js/
│   ├── css/
│   └── img/
└── admin/              ← 管理后台 SPA（独立路径）
    ├── index.html
    └── assets/
```

## 缓存策略

| 路径              | 说明                  | Cache-Control                        |
|-------------------|-----------------------|--------------------------------------|
| `/assets/**`      | 带 hash 的 JS/CSS/图片 | `max-age=604800, immutable`（7天）   |
| `/index.html`     | 前端入口              | `no-store`（不缓存）                 |
| `/sw.js`          | Service Worker        | `no-store`（不缓存）                 |
| `/workbox-*.js`   | Workbox 脚本          | `no-store`（不缓存）                 |
| `/admin/**`       | 管理后台所有资源      | `no-store`（不缓存）                 |
| `/uploads/**`     | 用户上传文件          | `no-store`（不缓存）                 |

> **为什么 assets 可以长期缓存？**  
> Vite 构建时会将文件内容的 hash 注入文件名（如 `main-abc123.js`），内容不变则文件名不变，内容改变则文件名随之改变，因此可以安全地设置长期缓存，不会出现缓存脏数据问题。

## SPA 路由支持（Vue Router History 模式）

后端 `WebConfig.java` 配置了 `PathResourceResolver`，当请求路径不对应实际文件时，自动 fallback 到 `index.html`，由 Vue Router 在前端解析路由。

```
GET /app/dashboard  → 无实际文件 → fallback → /index.html → Vue Router 渲染
GET /login          → 无实际文件 → fallback → /index.html → Vue Router 渲染
GET /admin/users    → 无实际文件 → fallback → /admin/index.html → Vue Router 渲染
```

**注意**：`/api/**` 开头的请求不走静态资源 handler，直接交给 Spring MVC Controller 处理。

## 一键部署步骤

### 方式一：使用 deploy.bat（推荐）

```bash
# 在项目根目录下双击或执行
deploy.bat
```

脚本会自动完成：
1. 调用 `scripts/构建前端应用.ps1` 安装依赖并构建前端
2. 调用 `scripts/同步静态资源.ps1` 清理并同步前端 `dist` 到 `static`
3. 可选调用 `scripts/构建前端应用.ps1` 构建 admin-vue
4. 再次调用 `scripts/同步静态资源.ps1 -AdminOnly` 清理并同步管理后台 `dist`
5. `mvn clean package -DskipTests` 打包 JAR

### 方式二：手动步骤

```powershell
# 1. 构建前端
powershell -ExecutionPolicy Bypass -File scripts\构建前端应用.ps1 -ProjectDir frontend-vue -Label 前端用户端

# 2. 回到项目根目录并同步静态资源
powershell -ExecutionPolicy Bypass -File scripts\同步静态资源.ps1

# 3. 如需管理后台，先构建再同步
powershell -ExecutionPolicy Bypass -File scripts\构建前端应用.ps1 -ProjectDir admin-vue -Label 管理后台
powershell -ExecutionPolicy Bypass -File scripts\同步静态资源.ps1 -AdminOnly

# 4. 打包后端
cd backend
mvn clean package -DskipTests
```

### 方式三：仅同步静态资源

当 `frontend-vue/dist` 或 `admin-vue/dist` 已经准备好时，可以单独执行同步脚本，而不必重新运行完整部署流程。

```powershell
# 仅同步用户前端
powershell -ExecutionPolicy Bypass -File scripts\同步静态资源.ps1

# 仅同步管理后台
powershell -ExecutionPolicy Bypass -File scripts\同步静态资源.ps1 -AdminOnly
```

### 方式四：仅执行前端构建

```powershell
# 构建用户前端
powershell -ExecutionPolicy Bypass -File scripts\构建前端应用.ps1 -ProjectDir frontend-vue -Label 前端用户端

# 构建管理后台
powershell -ExecutionPolicy Bypass -File scripts\构建前端应用.ps1 -ProjectDir admin-vue -Label 管理后台
```

## WebConfig.java 关键配置说明

```java
// ① 管理后台（优先注册，不缓存，SPA fallback 到 admin/index.html）
registry.addResourceHandler("/admin", "/admin/", "/admin/**")
        .addResourceLocations("classpath:/static/admin/")
        .setCacheControl(CacheControl.noStore())
        ...

// ② assets/（长期缓存，immutable）
registry.addResourceHandler("/assets/**", "/dist/assets/**")
        .addResourceLocations("classpath:/static/dist/assets/")
        .setCacheControl(CacheControl.maxAge(7, TimeUnit.DAYS).immutable());

// ③ Service Worker 相关（不缓存）
registry.addResourceHandler("/sw.js", "/registerSW.js", "/workbox-*.js",
        "/dist/sw.js", "/dist/registerSW.js", "/dist/workbox-*.js")
        .addResourceLocations("classpath:/static/dist/")
        .setCacheControl(CacheControl.noStore());

// ④ 用户前端（SPA fallback 到 dist/index.html，不缓存）
registry.addResourceHandler("/**")
        .addResourceLocations("classpath:/static/dist/")
        .setCacheControl(CacheControl.noStore())
        ...
```

## 常见问题

### Q: 刷新页面返回 404？
**A**: 确认 `WebConfig.java` 中的 `/**` handler 有 SPA fallback 逻辑（return `index.html`），且后端已重新打包。

### Q: JS/CSS 更新后浏览器还用旧缓存？
**A**: 检查 `vite.config.js` 中 `assetFileNames` 是否包含 `[hash]`，确保文件名随内容变化。

### Q: API 请求被当作静态资源处理？
**A**: `PathResourceResolver` 中有 `if (resourcePath.startsWith("api/")) return null;` 的检查，确保此代码存在。

### Q: admin 目录构建后不生效？
**A**: 管理后台需单独构建并复制到 `static/admin/`，运行 `deploy.bat` 并选择 `y` 构建管理后台即可。

### Q: 部署后 `/assets/**` 的 JS/CSS 返回 403？
**A**: 这通常不是前端打包错误，而是宝塔/Nginx 层拦截了静态资源。当前构建产物中的 [`index.html`](E:/Project/LearnSphere%20%20AI/backend/src/main/resources/static/dist/index.html) 会请求绝对路径 `/assets/...`，而后端 [`WebConfig.java`](E:/Project/LearnSphere%20%20AI/backend/src/main/java/com/learnsphere/config/WebConfig.java) 已支持 `/assets/**` 和 `/dist/assets/**`。如果浏览器看到的是 `403` 而不是 `404`，优先检查：

- Nginx 是否把整个站点都反代到 `127.0.0.1:8080`，而不是只代理 `/api`
- 宝塔站点是否开启了防盗链或静态资源访问限制
- 是否有额外 `location /assets/` 抢在反代之前处理请求

推荐先在服务器本机验证：

```bash
curl -I http://127.0.0.1:8080/assets/js/index-xxx.js
```

如果本机访问是 `200`，公网域名访问是 `403`，问题就在 Nginx/宝塔层。
