# 本地发版与宝塔上线检查清单

本文档用于发版前和宝塔上线后的快速核对，适合当前项目的常见路径：本地测试 -> 本地构建 -> Linux 宝塔部署。

## 一、发版前本地检查

### 配置检查

- [ ] 本地 MySQL 可连接
- [ ] 本地 Redis 可连接
- [ ] `AI_API_KEY` 已配置
- [ ] 如启用 Jasypt，本地已提供 `JASYPT_PASSWORD`
- [ ] 前台 `.env` 配置正确
- [ ] 管理后台 `.env` 配置正确

### 功能检查

- [ ] 用户端首页可正常打开
- [ ] 管理后台可正常打开
- [ ] 登录接口正常
- [ ] 关键列表页可正常加载
- [ ] Redis 相关功能正常
- [ ] AI 相关接口能正常返回

### 文档与配置边界检查

- [ ] 没有把真实密码、AI Key 写回源码
- [ ] `application-secret.properties` 未提交到仓库
- [ ] 本次使用的环境变量或外置配置来源明确

## 二、本地构建检查

### 前端与后台构建

- [ ] 已执行 `npm run build:frontend` 或 `npm run release:local`
- [ ] 如本次涉及管理后台，已执行 `npm run build:admin` 或 `npm run release:local:admin`
- [ ] 如采用合包发布，已完成静态资源同步

### 后端打包

- [ ] 已执行 `mvn clean package -DskipTests` 或统一发版命令
- [ ] 目标 Jar 已生成
- [ ] 本次准备上传的产物路径已确认

### 可选脚本入口

当前可直接复用：

- `npm run release:local`
- `npm run release:local:admin`
- `deploy.bat`
- `scripts/构建前端应用.ps1`
- `scripts/同步静态资源.ps1`
- `scripts/本地发版准备.ps1`

## 三、上传宝塔前检查

- [ ] 服务器已安装 JDK 17
- [ ] 服务器 MySQL 已准备好
- [ ] 服务器 Redis 已准备好
- [ ] Nginx 可正常配置反向代理
- [ ] 如需语音能力，服务器 Python 环境已准备好
- [ ] 外置配置目录路径已确认
- [ ] `application-secret.properties` 或环境变量已准备好

## 四、宝塔上线检查

### Java 项目配置

- [ ] 项目 Jar 路径正确
- [ ] `--spring.profiles.active=prod` 已配置
- [ ] 如使用外置配置，`--spring.config.additional-location` 路径正确且以 `/` 结尾
- [ ] JVM 参数已包含必要的编码/Jasypt 配置

### Nginx 配置

- [ ] `/api/` 已转发到后端端口
- [ ] 前台路由刷新正常
- [ ] 管理后台路由刷新正常

## 五、上线后验收

- [ ] 用户端首页可访问
- [ ] 管理后台可访问
- [ ] 登录成功
- [ ] 关键接口无 4xx/5xx 异常
- [ ] 宝塔 Java 项目状态正常
- [ ] 宝塔日志无明显启动报错
- [ ] 数据库连接正常
- [ ] Redis 连接正常
- [ ] AI 接口正常
- [ ] 如启用语音，本地语音能力正常

## 六、回滚前提

建议至少保留：

- 上一个稳定 Jar 包
- 上一个稳定配置文件副本
- 本次变更对应的文档入口
- 本次上线前后的关键日志记录
