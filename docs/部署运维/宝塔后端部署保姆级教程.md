# LearnSphere AI 宝塔后端部署保姆级教程

本文档用于 Linux 宝塔面板部署后端服务，默认前提是你已经在本地完成基本联调，并确认数据库、Redis、AI 接口可正常工作。

---

## 1. 环境准备

在宝塔面板的「软件商店」中确认以下组件可用：

1. **JDK 17**
2. **MySQL 8.0+**
3. **Redis 6.0+**
4. **Nginx**
5. **Python 3.10+**：仅当需要本地语音能力时安装

---

## 2. 数据库初始化

1. 创建数据库 `learnsphere_ai`
2. 字符集建议使用 `utf8mb4`
3. 导入项目根目录下的 `backend/learnsphere_ai_production.sql`
4. 创建一个实际可用的数据库账号，并授予该库权限

说明：账号和密码请按服务器实际环境创建，不要直接复用历史文档里的默认示例。

---

## 3. 本地打包与上传

### 3.1 本地打包后端

在本地 `backend` 目录执行：

```bash
mvn clean package -DskipTests
```

### 3.2 上传服务器文件

在服务器创建目录，例如：

```bash
/www/wwwroot/learnsphere-backend/
```

上传以下文件：

- `target/learnsphere-ai-backend-1.0.0.jar`
- `requirements-voice.txt`（如果需要语音功能）

---

## 4. 外置配置

建议在服务器创建配置目录：

```bash
/www/wwwroot/learnsphere-backend/config/
```

然后创建：

```properties
application-secret.properties
```

可参考模板：

- [backend/src/main/resources/application-secret.properties.example](E:/Project/LearnSphere%20%20AI/backend/src/main/resources/application-secret.properties.example)
- [环境变量与启动说明.md](E:/Project/LearnSphere%20%20AI/docs/%E9%83%A8%E7%BD%B2%E8%BF%90%E7%BB%B4/%E7%8E%AF%E5%A2%83%E5%8F%98%E9%87%8F%E4%B8%8E%E5%90%AF%E5%8A%A8%E8%AF%B4%E6%98%8E.md)

常见示例：

```properties
spring.datasource.username=your-db-username
spring.datasource.password=your-db-password
ai.api-key=your-ai-api-key
# spring.redis.password=your-redis-password
```

如启用了 Jasypt，请同时准备环境变量或 JVM 参数中的 `JASYPT_PASSWORD`。

---

## 5. 宝塔 Java 项目配置

进入「Java项目管理器」->「添加项目」：

- **项目 Jar 路径**: `/www/wwwroot/learnsphere-backend/learnsphere-ai-backend-1.0.0.jar`
- **项目端口**: `8080`
- **JVM 参数**:

```text
-Xms256M -Xmx1024M -Dfile.encoding=UTF-8 -DJASYPT_PASSWORD=your-jasypt-password
```

- **程序参数**:

```text
--spring.profiles.active=prod --spring.config.additional-location=file:/www/wwwroot/learnsphere-backend/config/
```

注意：`--spring.config.additional-location` 必须以 `/` 结尾。

---

## 6. 语音依赖安装

仅当需要本地语音能力时执行：

```bash
cd /www/wwwroot/learnsphere-backend
pip3 install -r requirements-voice.txt
```

如果服务器只做普通学习功能和管理后台，不启用本地语音，可先不安装这部分依赖。

---

## 7. Nginx 反向代理

在宝塔网站配置中增加 `/api/` 代理：

```nginx
location ^~ /api/ {
    proxy_pass http://127.0.0.1:8080/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

---

## 8. 验收清单

- 宝塔 Java 项目状态为“运行中”
- `https://你的域名/api/health/check` 可访问
- 数据库无 `Access denied` 错误
- Redis 连接正常
- AI 接口可正常返回
- 如启用语音，本地语音接口可正常工作

---

## 9. 常见问题

- **502 Bad Gateway**：先检查宝塔 Java 项目日志
- **配置未生效**：检查 `--spring.config.additional-location` 是否正确且以 `/` 结尾
- **数据库认证失败**：检查外置 `application-secret.properties` 或环境变量是否与数据库实际账号一致
- **语音报错**：检查 Python 依赖是否已安装
