# 移动端音频播放问题排查指南

## 问题描述

部署后移动端无法播放听力音频，可能表现为：
- 点击播放按钮无反应
- 音频加载失败
- 提示"音频播放失败"
- 自动切换到本地 TTS 但仍无法播放

## 常见原因

### 1. 网络连接问题

**症状：**
- 音频请求超时
- 控制台显示 `net::ERR_CONNECTION_TIMED_OUT`

**排查步骤：**
```bash
# 在移动端浏览器中访问
http://你的服务器IP:8080/api/tts/edge

# 应该返回 400 Bad Request（因为没有传参数），而不是超时或无法连接
```

**解决方案：**
- 检查服务器防火墙是否开放 8080 端口
- 检查宝塔面板安全组规则
- 确认移动端和服务器在同一网络或服务器有公网 IP

### 2. CORS 跨域问题

**症状：**
- 控制台显示 `CORS policy` 错误
- 提示 `Invalid CORS request`

**排查步骤：**
```javascript
// 在移动端浏览器控制台执行
fetch('/api/tts/edge', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ text: 'test', voice: 'en-US-JennyNeural', rate: 1.0 })
})
.then(r => console.log('Status:', r.status))
.catch(e => console.error('Error:', e))
```

**解决方案：**
- 确认 `CorsConfig.java` 中包含你的局域网 IP 段
- 检查 `application.yml` 中的 `app.cors.allowed-origins` 配置
- 重启后端服务使配置生效

### 3. 音频自动播放限制

**症状：**
- 首次点击播放无反应
- 控制台显示 `play() failed because the user didn't interact with the document first`

**排查步骤：**
- 检查是否在用户交互（点击）后才调用播放
- 查看 `primeMobileAudioPlayback` 是否被正确调用

**解决方案：**
- 确保音频播放在用户点击事件中触发
- 使用静音音频预先解锁音频上下文（已在代码中实现）

### 4. Edge TTS 服务异常

**症状：**
- 后端日志显示 `Edge TTS native synthesis failed`
- 返回 500 错误

**排查步骤：**
```bash
# 查看后端日志
tail -f /www/server/tomcat/logs/catalina.out

# 或在宝塔面板查看应用日志
```

**解决方案：**
- 检查服务器网络是否能访问 Edge TTS 服务
- 查看是否有防火墙拦截出站连接
- 考虑使用代理或 VPN

### 5. 静态资源路径问题

**症状：**
- 音频 Blob URL 创建成功但无法播放
- 控制台显示 404 或 403 错误

**排查步骤：**
```bash
# 检查静态资源目录结构
ls -la backend/src/main/resources/static/dist/

# 确认 index.html 和 assets 目录存在
```

**解决方案：**
- 重新构建前端：`npm run build:frontend`
- 确认 `WebConfig.java` 中的路径映射正确
- 检查 `vite.config.js` 中的 `outDir` 配置

## 使用诊断工具

### 开发环境诊断

访问调试页面（仅开发模式可用）：
```
http://localhost:5173/app/audio-debug
```

或在移动端：
```
http://你的电脑局域网IP:5173/app/audio-debug
```

### 生产环境诊断

1. **查看浏览器控制台**
   - Chrome Android: 在地址栏输入 `chrome://inspect`
   - Safari iOS: 连接 Mac，使用 Safari 开发者工具

2. **查看后端日志**
   ```bash
   # 实时查看日志
   tail -f /www/server/tomcat/logs/catalina.out | grep -i "tts\|audio"
   ```

3. **测试 API 端点**
   ```bash
   # 使用 curl 测试
   curl -X POST http://你的服务器IP:8080/api/tts/edge \
     -H "Content-Type: application/json" \
     -d '{"text":"Hello world","voice":"en-US-JennyNeural","rate":1.0}' \
     --output test.mp3

   # 检查生成的文件
   file test.mp3
   # 应该显示: test.mp3: Audio file with ID3 version 2.3.0
   ```

## 预防措施

### 1. 配置检查清单

部署前确认：
- [ ] CORS 配置包含所有需要的域名/IP
- [ ] 防火墙开放必要端口（8080）
- [ ] 静态资源正确构建到 `backend/src/main/resources/static/dist/`
- [ ] Edge TTS 服务可访问（测试网络连通性）

### 2. 监控配置

添加音频播放监控：
```yaml
# prometheus-alert-rules.yml
- alert: HighAudioPlaybackFailureRate
  expr: rate(audio_playback_errors_total[5m]) > 0.1
  for: 5m
  labels:
    severity: warning
  annotations:
    summary: "音频播放失败率过高"
```

### 3. 降级策略

当前已实现的降级策略：
1. Edge TTS 失败 → 自动切换到浏览器原生 TTS
2. 网络超时 → 缩短超时时间，快速降级
3. 移动端优化 → 预先解锁音频上下文

## 常见错误代码

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `net::ERR_CONNECTION_REFUSED` | 后端服务未启动 | 启动后端服务 |
| `net::ERR_CONNECTION_TIMED_OUT` | 网络不通或防火墙拦截 | 检查网络和防火墙 |
| `403 Invalid CORS request` | CORS 配置不正确 | 更新 CORS 配置 |
| `404 Not Found` | 静态资源路径错误 | 检查 WebConfig 配置 |
| `Empty audio payload` | Edge TTS 返回空数据 | 检查网络和 Edge TTS 服务 |
| `play() failed` | 浏览器自动播放限制 | 确保在用户交互中播放 |
