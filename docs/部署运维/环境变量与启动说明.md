# 环境变量与启动说明

## 适用范围

本文档用于当前仓库的非 Docker 场景，重点覆盖两类使用方式：

- 本地开发与本地联调测试
- Linux 服务器上的宝塔部署

涉及模块：

- 前端应用 `frontend-vue`
- 管理后台 `admin-vue`
- Spring Boot 后端 `backend`

## 一、前端环境变量

### 1. 前端应用

参考文件：[frontend-vue/.env.example](E:/Project/LearnSphere%20%20AI/frontend-vue/.env.example)

```env
VITE_API_URL=/api
```

说明：

- 本地联调建议保持 `/api`，通过 Vite 代理转发到本地后端。
- 如果前端独立部署，也可以改为完整后端地址，例如 `http://127.0.0.1:8080/api`。

### 2. 管理后台

参考文件：[admin-vue/.env.example](E:/Project/LearnSphere%20%20AI/admin-vue/.env.example)

```env
VITE_API_URL=/api
```

## 二、后端配置来源

后端主配置位于：

- [backend/src/main/resources/application.yml](E:/Project/LearnSphere%20%20AI/backend/src/main/resources/application.yml)
- [backend/src/main/resources/application-prod.yml](E:/Project/LearnSphere%20%20AI/backend/src/main/resources/application-prod.yml)
- [backend/src/main/resources/application-secret.properties.example](E:/Project/LearnSphere%20%20AI/backend/src/main/resources/application-secret.properties.example)

当前约定：

- 非敏感通用配置保留在 `application.yml`
- 生产覆盖项保留在 `application-prod.yml`
- 数据库、Redis、AI Key 等敏感信息通过环境变量或 `application-secret.properties` 注入
- `application-secret.properties` 已被 `.gitignore` 忽略，不应提交到仓库

### 1. 推荐环境变量

```env
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=learnsphere_ai
DB_USERNAME=your-db-username
DB_PASSWORD=your-db-password
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DATABASE=0
AI_API_KEY=your-ai-api-key
AI_MODEL=qwen-turbo
CORS_ALLOWED_ORIGINS=http://localhost:5173,http://localhost:5174
JASYPT_PASSWORD=your-jasypt-password
```

### 2. 可选 secret 文件

如需文件注入，可复制以下模板：

- [backend/src/main/resources/application-secret.properties.example](E:/Project/LearnSphere%20%20AI/backend/src/main/resources/application-secret.properties.example)

复制后生成：

```properties
backend/src/main/resources/application-secret.properties
```

常见内容示例：

```properties
spring.datasource.username=your-db-username
spring.datasource.password=your-db-password
ai.api-key=your-ai-api-key
# spring.redis.password=your-redis-password
```

## 三、本地测试建议路径

如果你的主要使用方式是本地测试，建议按下面顺序：

### 1. 准备本地依赖

- 启动 MySQL
- 启动 Redis
- 准备 AI Key

### 2. 安装前端依赖

在仓库根目录执行：

```powershell
npm install
```

### 3. 启动前端应用

```powershell
npm run dev:frontend
```

### 4. 启动管理后台

```powershell
npm run dev:admin
```

### 5. 启动后端

在 `backend` 目录按现有 Java 工程方式启动 Spring Boot。启动前确认：

- MySQL 已可连接
- Redis 已可连接
- `AI_API_KEY` 或 `application-secret.properties` 已配置
- 如启用 Jasypt，加密口令 `JASYPT_PASSWORD` 已注入

### 6. 本地联调建议

- 用户端：`http://localhost:5173`
- 管理后台：`http://localhost:5174`
- 后端：`http://localhost:8080`

建议本地优先验证：

- 登录与鉴权
- 普通 API 请求
- 管理后台主要列表页
- AI 相关接口是否可正常返回
- Redis 相关配额/缓存逻辑

## 四、Linux 宝塔部署建议路径

如果目标是 Linux 服务器上的宝塔部署，建议按下面顺序：

### 1. 本地先完成构建验证

在仓库根目录执行：

```powershell
npm run build:frontend
npm run build:admin
```

后端按现有 Java 工程方式完成打包，例如：

```bash
mvn clean package -DskipTests
```

### 2. 服务器准备

建议在宝塔中准备：

- JDK 17
- MySQL 8.0+
- Redis 6.0+
- Nginx
- Python 3.10+（仅当需要本地语音能力时）

### 3. 配置注入方式

宝塔生产环境优先采用以下两种方式之一：

- 环境变量注入
- 外置 `application-secret.properties` 文件

不再建议把真实密码、AI Key 写回源码内配置文件。

### 4. Java 项目启动关键项

常见程序参数：

```text
--spring.profiles.active=prod --spring.config.additional-location=file:/你的部署目录/config/
```

常见 JVM 参数：

```text
-Xms256M -Xmx1024M -Dfile.encoding=UTF-8 -DJASYPT_PASSWORD=your-jasypt-password
```

### 5. Nginx 代理建议

统一把 `/api/` 转发到 Spring Boot 服务，例如 `http://127.0.0.1:8080/`，前台和后台都走同一后端。

## 五、构建入口

当前仓库已经支持从根目录统一触发前后台构建：

```powershell
npm run build:frontend
npm run build:admin
npm run build
```

如果需要同步静态资源到后端，可继续使用现有脚本：

- `deploy.bat`
- `scripts/构建前端应用.ps1`
- `scripts/同步静态资源.ps1`

## 六、交接检查清单

新环境接手前，至少确认以下事项：

- 前台和后台 `.env` 已按实际地址填写
- MySQL、Redis 连接信息不再依赖源码内明文默认值
- `AI_API_KEY` 已通过环境变量或 secret 文件注入
- `application-secret.properties` 未提交到 Git
- 生产环境已提供 `JASYPT_PASSWORD`
- 宝塔启动参数与配置文件目录已对应到实际服务器路径
