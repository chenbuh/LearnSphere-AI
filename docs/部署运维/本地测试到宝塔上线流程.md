# 本地测试到宝塔上线流程

本文档面向当前项目的常用路径：

1. 先在本地完成前后端联调
2. 再在本地完成构建与打包
3. 最后部署到 Linux 服务器的宝塔环境

不使用 Docker。

## 一、推荐总流程

### 阶段 1：本地联调验证

先确认以下内容在本地可用：

- MySQL 可连接
- Redis 可连接
- `AI_API_KEY` 已配置
- 用户端能正常访问
- 管理后台能正常访问
- 关键接口无明显报错

推荐入口：

```powershell
npm install
npm run dev:frontend
npm run dev:admin
```

后端按当前 Java 工程方式在 `backend` 目录启动。

### 阶段 2：本地构建与打包

本地联调通过后，再进行构建和静态资源合包。

推荐三种方式。

#### 方式 A：使用根目录统一命令

仅构建用户端并完成后端打包：

```powershell
npm run release:local
```

如果本次包含管理后台，一并执行：

```powershell
npm run release:local:admin
```

这两个命令会调用 `scripts/本地发版准备.ps1`，串起前端构建、静态资源同步和后端打包。

#### 方式 B：直接使用一键脚本

```powershell
deploy.bat
```

该脚本会按顺序执行：

1. 构建 `frontend-vue`
2. 同步用户端静态资源到 `backend/src/main/resources/static`
3. 询问是否构建 `admin-vue`
4. 如选择构建，则同步管理后台静态资源到 `backend/src/main/resources/static/admin`
5. 执行 `backend` 的 Maven 打包

#### 方式 C：分步执行

```powershell
powershell -ExecutionPolicy Bypass -File scripts\构建前端应用.ps1 -ProjectDir frontend-vue -Label 前端用户端
powershell -ExecutionPolicy Bypass -File scripts\同步静态资源.ps1
powershell -ExecutionPolicy Bypass -File scripts\构建前端应用.ps1 -ProjectDir admin-vue -Label 管理后台
powershell -ExecutionPolicy Bypass -File scripts\同步静态资源.ps1 -AdminOnly
```

然后在 `backend` 目录执行：

```bash
mvn clean package -DskipTests
```

### 阶段 3：上传宝塔服务器

建议上传以下内容：

- 后端 Jar 包
- 如需要本地语音功能，则上传 `requirements-voice.txt`
- 如采用外置配置，则在服务器创建 `config/application-secret.properties`

### 阶段 4：宝塔侧配置

在宝塔侧重点确认：

- Java 项目使用 JDK 17
- MySQL、Redis 已就绪
- 外置配置或环境变量已正确注入
- 程序参数使用 `--spring.profiles.active=prod`
- 如使用外置配置目录，`--spring.config.additional-location` 路径正确且以 `/` 结尾
- Nginx 已将 `/api/` 转发到后端端口

### 阶段 5：上线验收

至少检查：

- 用户端首页可访问
- 管理后台可访问
- 登录接口正常
- 关键列表页可加载
- Redis 相关功能正常
- AI 接口能正常返回
- 宝塔项目日志无明显启动异常

## 二、你当前最适合的使用方式

如果你主要是在本地测试，再部署到 Linux 宝塔，建议长期按下面的节奏：

1. 本地先做日常开发和联调
2. 需要发版时，在本地运行统一命令或构建脚本完成打包
3. 将通过本地验证的产物上传到宝塔
4. 宝塔只负责运行和代理，不承担开发阶段排错

这样做的好处是：

- 配置问题更容易定位
- 宝塔环境更干净
- 上线动作更稳定
- 不容易把“本地未验证”的问题带到服务器上
