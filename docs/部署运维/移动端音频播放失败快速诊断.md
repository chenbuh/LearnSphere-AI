# 移动端音频播放失败快速诊断指南

## 问题现象

服务器部署环境下，Android 手机浏览器无明显错误提示，但音频无法播放。

## 快速诊断步骤

### 步骤 1: 使用移动端测试工具

访问测试页面（需要先构建前端）：

```
http://你的服务器IP:8080/app/mobile-audio-test
```

按照页面提示依次点击三个按钮：
1. 解锁音频上下文
2. 获取音频
3. 播放音频

查看每一步的日志输出，定位具体失败环节。

### 步骤 2: 检查后端服务

在服务器上执行：

```bash
# 检查后端是否正常运行
curl http://localhost:8080/api/health

# 测试 Edge TTS API
curl -X POST http://localhost:8080/api/tts/edge \
  -H "Content-Type: application/json" \
  -d '{"text":"test","voice":"en-US-JennyNeural","rate":1.0}' \
  --output test.mp3

# 检查生成的文件
file test.mp3
# 应该显示: test.mp3: Audio file with ID3 version 2.3.0
```

### 步骤 3: 检查网络连接

在手机浏览器中访问：

```
http://你的服务器IP:8080/api/tts/edge
```

- 如果显示 400 Bad Request → 后端正常，网络通畅
- 如果显示连接超时 → 网络问题或防火墙拦截
- 如果显示 404 → 后端路由配置问题

### 步骤 4: 查看后端日志

```bash
# 实时查看日志
tail -f /www/server/tomcat/logs/catalina.out | grep -i "tts\|edge"

# 或查看最近的错误
tail -100 /www/server/tomcat/logs/catalina.out | grep -i "error\|exception"
```

## 常见问题和解决方案

### 问题 1: 音频上下文未解锁

**症状：** 点击播放按钮无反应，控制台无错误

**原因：** 移动端浏览器要求用户交互后才能播放音频

**解决方案：**
- 已在代码中添加自动解锁机制
- 确保用户点击播放按钮（不是自动播放）
- 重新构建前端并部署

### 问题 2: Edge TTS 服务无法访问

**症状：** 测试工具显示"音频获取失败"

**原因：** 服务器无法连接到 Edge TTS 服务

**解决方案：**

```bash
# 在服务器上测试网络连接
curl -I https://speech.platform.bing.com

# 如果无法访问，检查防火墙或使用代理
```

### 问题 3: CORS 跨域问题

**症状：** 控制台显示 CORS 错误

**原因：** 后端 CORS 配置不包含当前访问域名

**解决方案：**

检查 `application.yml` 中的 CORS 配置：

```yaml
app:
  cors:
    allowed-origins: http://localhost:5173,http://192.168.*:*,http://你的服务器IP:*
```

### 问题 4: 音频格式不支持

**症状：** 音频获取成功但播放失败

**原因：** 浏览器不支持音频格式

**解决方案：**

检查 Edge TTS 返回的音频格式：

```bash
# 查看音频文件信息
ffprobe test.mp3

# 应该是 MP3 格式，大多数浏览器都支持
```

### 问题 5: 静态资源 404

**症状：** 页面加载不完整，部分 JS/CSS 404

**原因：** 前端构建产物未正确部署

**解决方案：**

```bash
# 重新构建前端
npm run build:frontend

# 重新打包后端
cd backend
mvn clean package -DskipTests

# 上传新的 JAR 文件并重启
```

## 详细诊断流程

### 1. 检查浏览器控制台

在手机浏览器中打开开发者工具（如果支持）：

**Chrome Android:**
1. 在电脑上打开 Chrome
2. 访问 `chrome://inspect`
3. 连接手机并选择对应页面

**查找关键日志：**
- `[TTS]` - TTS 音频获取相关
- `[AudioPlayer]` - 音频播放器相关
- `[MobilePlayback]` - 移动端播放相关

### 2. 检查网络请求

在 Network 标签中查看：

```
/api/tts/edge
```

**正常响应：**
- Status: 200
- Content-Type: audio/mpeg
- Size: > 0 KB

**异常响应：**
- Status: 500 → 后端错误
- Status: 404 → 路由配置错误
- Status: 403 → CORS 或认证问题
- Timeout → 网络或服务器问题

### 3. 检查音频元素

在控制台执行：

```javascript
// 检查音频元素
document.querySelectorAll('audio')

// 手动测试播放
const audio = new Audio('http://你的服务器IP:8080/api/tts/edge')
audio.play().then(() => console.log('播放成功')).catch(e => console.error('播放失败', e))
```

## 如果仍然失败

### 收集诊断信息

1. **移动端测试工具的完整日志**
   - 截图或复制所有日志内容

2. **浏览器控制台日志**
   - 包含所有 `[TTS]`, `[AudioPlayer]`, `[MobilePlayback]` 前缀的日志

3. **后端日志**
   ```bash
   tail -200 /www/server/tomcat/logs/catalina.out > backend-logs.txt
   ```

4. **网络请求详情**
   - Chrome DevTools Network 标签的截图
   - 包含 `/api/tts/edge` 请求的详细信息

5. **设备信息**
   - 手机型号
   - Android 版本
   - 浏览器版本

### 临时解决方案

如果 Edge TTS 持续失败，可以强制使用浏览器原生 TTS：

在 `AudioPlayer.vue` 中修改：

```javascript
// 临时禁用 Edge TTS，直接使用原生 TTS
async function playFallbackAudio() {
  const normalizedText = (props.audioText || '').trim()
  if (normalizedText) {
    playNativeTTS(normalizedText)
    return true
  }
  return false
}
```
