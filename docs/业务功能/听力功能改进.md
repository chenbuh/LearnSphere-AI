# 听力训练题目数量修改方案

## 现状问题
- 用户选择"5篇"，但只生成3道题
- 当前实现：1篇听力材料 = 3道左右的题目

## 解决方案

### 选项1：按题目数量（简单，推荐）
将"篇章数量"概念改为"题目数量"
- 用户选择5 = 生成5道题目
- 都基于同一篇听力材料

**修改点：**
1. 前端：将"篇章数量"改为"题目数量"
2. 后端：修改prompt，让AI生成指定数量的题目

### 选项2：按真实考试格式（复杂）
保持"篇章"概念，每篇3-5题
- 用户选择5篇 = 5段音频 + 15-25道题
- 需要重构数据结构支持多段音频

**修改点：**
1. 数据结构：支持多段音频passages
2. 前端：支持播放不同段落的音频
3. 后端：生成多篇听力材料

## 推荐实施：选项1

修改文件：
1. ListeningView.vue - 改标签"题目数量"
2. AIGenerationServiceImpl.java - 修改prompt生成count数量的题目
