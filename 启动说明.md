# 🚀 启动说明

## 全栈开发环境启动

### 🎯 推荐方式：同时启动前后端
```bash
npm run fullstack
```

这个命令会同时启动：
- 🔵 **后端服务器** (Node.js + Express) - 端口8080
- 🟣 **前端构建工具** (Webpack + PostCSS) - 自动监听文件变化

### 🌟 高级启动方式（推荐）
```bash
npm run dev:complete
```

这个命令提供更丰富的功能：
- 彩色日志输出
- 端口占用检查
- 自动打开浏览器
- 详细的启动状态显示

### 📋 其他可用命令

| 命令 | 功能 |
|------|------|
| `npm run fullstack` | 基础全栈启动 |
| `npm run dev:complete` | 高级全栈启动（推荐） |
| `npm run dev` | 仅启动后端服务器 |
| `npm run dev:backend` | 仅启动后端服务器 |
| `npm run dev:frontend` | 仅启动前端构建工具 |
| `npm start` | 生产环境启动 |

## 🌐 访问地址

启动后可访问：
- **主应用**: http://localhost:8080/
- **AI演示**: http://localhost:8080/demo
- **API状态**: http://localhost:8080/api/health
- **学习统计**: http://localhost:8080/api/learning/stats

## 💡 开发提示

### 热重载功能
- ✅ 修改JavaScript文件 → 后端自动重启
- ✅ 修改CSS文件 → 自动重新编译
- ✅ 修改HTML文件 → 刷新浏览器即可看到变化

### 停止服务
- 在终端中按 `Ctrl + C` 停止所有服务

### 查看日志
- 后端日志：蓝色标识 `[BACKEND]`
- 前端CSS：紫色标识 `[FRONTEND]`
- 构建信息：不同颜色区分

## 🔧 故障排除

### 端口被占用
如果8080端口被占用：
```bash
# Windows
netstat -ano | findstr :8080
taskkill /PID <PID号> /F

# 或者修改端口
set PORT=3000
npm run fullstack
```

### 依赖问题
```bash
# 重新安装依赖
rm -rf node_modules package-lock.json
npm install
```

### 清理缓存
```bash
npm cache clean --force
```
